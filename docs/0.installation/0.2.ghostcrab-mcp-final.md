# GhostCrab MCP

GhostCrab MCP bridges MCP-compatible harnesses to mindBrain's structured memory.

---

## What is MCP?

The **Model Context Protocol** is an open standard that lets AI agents call typed, named tools exposed by external servers over a structured stdio interface. Any MCP-compatible client ‚Äî Claude Code, Cursor, Codex, OpenClaw ‚Äî discovers the server's tool catalog at startup and calls tools by name with JSON arguments. The protocol handles transport, schema validation, and error reporting; the server handles the logic. [Full specification: modelcontextprotocol.io](https://modelcontextprotocol.io)

---

## The problem without GhostCrab

An MCP agent without persistent structured memory operates in a flat context window. It knows what was said in the current session. When the session ends, that knowledge disappears.

More precisely, what it cannot do:

- Retrieve all entities of a given type, filtered by state, grouped by dimension ‚Äî across sessions
- Traverse typed relations between entities to find blockers, dependencies, or coverage gaps
- Resume a working context from exactly where a previous session stopped, with the same projection intact

Every session reconstructs these answers from conversation history. The cost is real: tokens consumed, latency added, consistency eroded. At scale ‚Äî dozens of agents, hundreds of sessions per day ‚Äî this is not a memory limitation. It is an architectural one.

GhostCrab adds what MCP alone cannot provide: a persistent, typed, queryable knowledge graph that survives sessions, agents, and restarts.

---

## What GhostCrab adds

Three capabilities mapped to three PostgreSQL extensions:

| Question | Without GhostCrab | With GhostCrab |
|---|---|---|
| *What are all entities of type X, filtered by Y, grouped by Z?* | Reconstruct from chat history | `ghostcrab_count` ‚Äî bitmap-indexed, < 1ms |
| *What does X depend on, 3 hops away?* | Not recoverable across sessions | `ghostcrab_traverse` ‚Äî typed graph, depth-controlled |
| *Where did the agent stop last session?* | Approximated from chat history | `ghostcrab_pack` ‚Äî packed projection, exact and resumable |

All three simultaneously, in a single PostgreSQL transaction.

## Personal (SQLite) vs Pro (PostgreSQL)

GhostCrab ships in two variants. Same 24 tools. Same MCP interface. Different engine.

| | **GhostCrab Personal** | **GhostCrab Pro** |
|---|---|---|
| **Engine** | SQLite | PostgreSQL + native extensions |
| **Install** | `npm install @mindflight/ghostcrab-personal-mcp` | Docker + native PostgreSQL image |
| **Setup time** | < 2 minutes, no Docker | 10‚Äì20 minutes, Docker required |
| **Concurrent agents** | Single agent, local | Multi-agent, concurrent writes |
| **Scale** | Up to ~1M entities | Up to 4.3B+ entities |
| **Native acceleration** | ‚Äî | pgfacets ¬∑ pgdgraph ¬∑ pgpragma |
| **BM25 search** | FTS via SQLite FTS5 | Native BM25 via pgfacets |
| **Graph traversal** | Recursive SQL | Native neighborhood via pgdgraph |
| **Projection packing** | SQL | Native pragma_pack_context |
| **License** | Apache 2.0 | Pro license |
| **Best for** | Personal agents, local dev, community integrations | Production workloads, team deployments, enterprise |

**Start with Personal.** It runs on any machine with Node.js installed, no additional infrastructure. Migrate to Pro when you need concurrent agents, multi-workspace isolation, or dataset scale beyond what SQLite handles comfortably.

## Install ‚Äî Personal (SQLite)

Zero-infrastructure install. One command, one process, no Docker.

```bash
npm install -g @mindflight/ghostcrab-personal-mcp
```

Verify the install:

```bash
ghostcrab-personal --version
ghostcrab-personal tools list
# ‚Üí 24 tools listed
```

The SQLite database is created automatically on first run at `~/.mindbrain/ghostcrab.db`. No configuration required.

---

## Install ‚Äî Pro (PostgreSQL)

Requires Docker. The native PostgreSQL image bundles `pgfacets`, `pgdgraph`, `pgpragma`, and `roaringbitmap` ‚Äî these are not available on standard PostgreSQL images.

```bash
git clone https://github.com/mitambo/ghostcrab.git ghostcrab
cd ghostcrab
cp .env.example .env
npm install

# Start the native PostgreSQL stack
docker compose -f docker/docker-compose.native.yml up -d --build postgres

# Migrate and build
npm run migrate
npm run build

# Start the MCP server
node dist/index.js
# ‚Üí GhostCrab MCP listening ‚Äî 24 tools registered
```

Verify native readiness:

```bash
ghostcrab tools list   # confirms 24 tools
# OR via MCP:
# ghostcrab_status ‚Üí runtime.native_readiness
```

## Connect to your MCP client

Add the server block to your client configuration. The path must be absolute.

**Claude Desktop** (`~/Library/Application Support/Claude/claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "ghostcrab": {
      "command": "ghostcrab-personal",
      "args": ["--mcp"]
    }
  }
}
```

*Pro variant ‚Äî replace `command` and `args` with:*
```json
"command": "node",
"args": ["/absolute/path/to/ghostcrab/dist/index.js"]
```

**Cursor** (Settings ‚Üí MCP Servers ‚Üí Add):

```json
{
  "mcpServers": {
    "ghostcrab": {
      "command": "ghostcrab-personal",
      "args": ["--mcp"]
    }
  }
}
```

**Codex** (`codex/ghostcrab-memory/SKILL.md`):

The Codex integration reads the intake contract from `ghostcrab-skills/shared/ONBOARDING_CONTRACT.md` and enforces the 5-question gate before any write operation. The config block is identical in structure ‚Äî point `args` to either the Personal binary or the Pro `dist/index.js`.

```json
{
  "mcpServers": {
    "ghostcrab": {
      "command": "ghostcrab-personal",
      "args": ["--mcp"]
    }
  }
}
```

Restart your client. Confirm the `ghostcrab_*` tools appear in the tool list before proceeding.

## 24 MCP tools

All tools are prefixed `ghostcrab_`. Grouped by subsystem. Tool input/output contracts are documented in `docs/mcp/tools-contract.md` in the repository.

### Facets ‚Äî 8 tools

Typed entity storage, BM25 full-text search, aggregated counts, hierarchical facet navigation.

| Tool | W/R | What it does | Key input fields |
|---|---|---|---|
| `ghostcrab_remember` | W | Insert a new fact into `mfo_facets` | `content`, `schema_id`, `facets` (typed dims), optional `embedding` |
| `ghostcrab_upsert` | W | Update-or-create current-state fact (idempotent by entity id) | `id`, `schema_id`, `facets` |
| `ghostcrab_search` | R | BM25 full-text search with optional facet filters | `query`, `mode` (bm25/exact), `filters`, `limit` |
| `ghostcrab_count` | R | Aggregated counts grouped by registered dimensions | `group_by` (array of dim names), `filters` |
| `ghostcrab_facet_tree` | R | Hierarchical facet tree for a dimension | `dimension`, `workspace_id` |
| `ghostcrab_schema_register` | W | Declare a new entity type with typed fields | `type`, `fields` (array of `{name, type}`) |
| `ghostcrab_schema_list` | R | List all registered entity schemas | `workspace_id` |
| `ghostcrab_schema_inspect` | R | Full field definition for a specific schema | `schema_id` |

### Graph ‚Äî 5 tools

Directed typed edges, multi-hop traversal, confidence-scored marketplace search.

| Tool | W/R | What it does | Key input fields |
|---|---|---|---|
| `ghostcrab_learn` | W | Upsert a graph node and/or a directed edge | `node` (`{id, node_type, label, properties}`) or `edge` (`{source, target, label, weight}`) |
| `ghostcrab_traverse` | R | Multi-hop traversal from a start node | `start`, `direction` (outbound/inbound/both), `depth`, `edge_labels` |
| `ghostcrab_marketplace` | R | Semantic graph search with confidence scoring | `query`, `limit`, `min_confidence`, `max_hops` |
| `ghostcrab_patch` | W | Apply a batch of graph operations in one transaction | `operations` (array of node/edge upserts and deletes) |
| `ghostcrab_coverage` | R | Coverage report ‚Äî gap nodes with decayed confidence | `workspace_id`, `min_confidence` |

### Projections ‚Äî 3 tools

Operational context, working views, resumable session state.

| Tool | W/R | What it does | Key input fields |
|---|---|---|---|
| `ghostcrab_project` | W | Declare or update a named working projection | `scope`, `content`, `proj_type`, `status`, `agent_id` |
| `ghostcrab_pack` | R | Pack and return current projection context | `query`, `agent_id`, `limit` ‚Äî native `pragma_pack_context` when available |
| `ghostcrab_status` | R | Full runtime snapshot: readiness, capabilities, facet delta status | ‚Äî (no required inputs) |

### Workspace ‚Äî 7 tools

Isolation scopes, human-gated DDL lifecycle, semantics export.

| Tool | W/R | What it does | Key input fields |
|---|---|---|---|
| `ghostcrab_workspace_create` | W | Create a workspace (idempotent) | `name`, `description` |
| `ghostcrab_workspace_list` | R | List all workspaces with live stats | ‚Äî |
| `ghostcrab_workspace_inspect` | R | Introspect Layer 1 semantics metadata | `workspace_id` |
| `ghostcrab_workspace_export_model` | R | Export workspace model to JSON consumer contract | `workspace_id` |
| `ghostcrab_ddl_propose` | W | Propose a DDL migration for human review | `sql`, `description`, optional `semantic_spec`, `sync_spec` |
| `ghostcrab_ddl_list_pending` | R | List pending, approved, and executed migrations | `workspace_id` |
| `ghostcrab_ddl_execute` | W | Execute an approved migration (DDL + triggers, one transaction) | `migration_id` |

### Geo ‚Äî 1 tool (optional)

Requires PostGIS. Returns a structured `geo_feature_not_available` error on standard deployments.

| Tool | W/R | What it does | Key input fields |
|---|---|---|---|
| `ghostcrab_query_geo` | R | Query geo-entities by bounding box or proximity | `lat`, `lng`, `radius_km` or `bbox` |

## First-Turn Intake Contract

When a user's first request is fuzzy ‚Äî *"help me model my CRM"*, *"set up a brain for my project"* ‚Äî GhostCrab enforces a mandatory intake sequence before any write tool is called.

The agent must ask exactly these five questions:

1. What is the domain? *(e.g. SaaS product, CRM pipeline, compliance knowledge base)*
2. What are the main entity types? *(5‚Äì10 to start)*
3. What are the primary relations between them?
4. What state or workflow needs to be tracked over time?
5. Are there existing data sources to import? *(CSV, API, DB dump)*

No call to `ghostcrab_remember`, `ghostcrab_learn`, or `ghostcrab_upsert` before the ontology is confirmed by the user. The first reply ends with: *"Ready. I'll wait for your answers before writing to the database."*

This contract is enforced identically across Claude Code, Cursor, and Codex via `ghostcrab-skills/shared/ONBOARDING_CONTRACT.md`. Skipping it produces orphaned nodes, non-canonical schemas, and a knowledge graph that becomes inconsistent within the first few sessions.

## Community integrations

GhostCrab Personal is the entry point for community-native deployments. Each framework integration ships as a standalone adapter that exposes mindBrain memory without requiring users to know the full tool surface.

- **[Claude Code](../integrations/claude-code.md)** ‚Äî native MCP client, zero-config with Personal; full workflow via SKILL.md
- **[Cursor](../integrations/cursor.md)** ‚Äî MCP client built into the editor; Personal binary drops in directly
- **[Codex](../integrations/codex.md)** ‚Äî explicit 5-question gate via SKILL.md, tested with Tier 1 models
- **[OpenClaw / AlphaClaw](../integrations/openclaw.md)** ‚Äî deploy AlphaClaw on Render, point at GhostCrab Personal or Pro
- **[CrewAI](../integrations/crewai.md)** ‚Äî `MindBrainMemoryTool` wrapping `ghostcrab_search`, `ghostcrab_remember`, `ghostcrab_traverse`
- **[n8n](../integrations/n8n.md)** ‚Äî community node `MindBrain Memory` for workflow agents; marketplace-installable

Each integration page documents the exact config block, the tool subset used, and the behavioral differences from the full 24-tool surface.
