# Installation

Get GhostCrab Perso running locally with SQLite and the `gcp brain` CLI.

---

## Prerequisites

- Node.js 18+
- A terminal
- No API key and no cloud account for the local SQLite path

---

## Install or run the package

Global install:

```bash
npm install -g @mindflight/ghostcrab-personal-mcp@latest
```

Reliable smoke check without global install:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp --help
```

The package pulls a platform-specific optional dependency for the native backend. If a postinstall message asks for binary permissions, run:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp authorize
```

If you install the package inside a project that already has a `package.json`, postinstall creates `./data/`, copies `.env` from the package example when missing, and adds documentation symlinks at the project root. To opt out:

```bash
GHOSTCRAB_SKIP_HOST_BOOTSTRAP=1 npm install @mindflight/ghostcrab-personal-mcp@latest
```

pnpm 10+ may ignore postinstall scripts by default. Approve the package build once, or install with:

```bash
pnpm add --allow-build=@mindflight/ghostcrab-personal-mcp @mindflight/ghostcrab-personal-mcp@latest
```

## Verify before wiring an IDE

Cursor, Codex, Claude, and other MCP hosts spawn a process from their MCP configuration. Confirm the CLI loads from this project first:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain up --help
```

Optional deeper check:

```bash
timeout 8 npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain up
```

This prints startup information, resolves the SQLite path, and then waits on stdio for MCP. Stop it with Ctrl+C if your shell does not exit through `timeout`.

---

## Wire your IDE

From the same project, run the setup command for your client:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain setup cursor --force
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain setup codex
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain setup claude
```

The generator chooses the most reliable launch form available: local absolute `node` plus package bin when locally installed, global `gcp` when available, or the `npx --package` fallback.

MCP clients should start:

```bash
gcp brain up
```

or the equivalent generated command. `gcp up` and `gcp start` are aliases for `gcp brain up`.

## Create a workspace

Create or register a workspace:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain workspace create my-app
```

List known workspaces:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain workspace list
```

Start MCP on that workspace:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain up --workspace my-app
```

## What "SQLite backend" means

GhostCrab Perso stores the local database in a SQLite file. In this starter path, the default project-local file is:

```text
./data/ghostcrab.sqlite
```

The exact path is current-working-directory dependent and can be confirmed by `gcp brain up` or `ghostcrab_status`.

- No standalone database server is required.
- No API key is required for local storage.
- The database is portable as a local file.
- The MCP surface remains the main ontology and query interface for agents.

This is the right backend for local development, single-agent workflows, and ontology prototyping.

## Schema and ontology commands

Use the current `gcp brain schema` command group to discover and manage ontology profiles:

```bash
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain schema --help
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain schema list
npx -y --package=@mindflight/ghostcrab-personal-mcp@latest gcp brain schema list --remote
```

Common subcommands:

```text
gcp brain schema list [--remote]
gcp brain schema pull <owner/name>
gcp brain schema show <owner/name>
gcp brain schema remove <owner/name>
```

`gcp ontologies <subcommand>` is an alias for the same schema command group.

This Perso starter kit does not rely on a separate legacy YAML loader command. Custom schema registration is done through MCP after model confirmation.

## SQLite vs PostgreSQL Pro

SQLite is the default for this guide. PostgreSQL Pro is for shared or production deployments that need a server-backed graph and native extension acceleration.

| | SQLite Perso | PostgreSQL Pro |
|---|---|---|
| Setup | Local file | PostgreSQL instance |
| Network | None for local storage | TCP connection to PG host |
| Concurrency | Local/single-user oriented | Multi-agent shared backend |
| Storage path | `./data/ghostcrab.sqlite` by default | Connection URL |
| Best for | Local development and prototyping | Shared production graphs |

Exact migration and Pro setup commands depend on the Pro package and deployment environment, so keep this guide focused on Perso.
