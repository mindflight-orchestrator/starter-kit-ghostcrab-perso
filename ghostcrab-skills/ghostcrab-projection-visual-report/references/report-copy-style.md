# Report Copy Style

Reports should feel like an expert audit note, not a raw database dump.

## Tone

- Direct, concrete, and business-readable.
- Short paragraphs.
- Numbers first, interpretation second.
- Avoid generic claims like "improve SEO" unless tied to evidence.

## Structure

1. One-line executive summary.
2. Timeline or KPI strip.
3. What changed.
4. What remains.
5. Evidence table.
6. Recommended actions.
7. Provenance.

## Wording Patterns

Good:

```text
The issue is no longer structural after A2. It can move from remediation to monitoring.
```

Good:

```text
A1 fixes the main pages but leaves a small residual cluster on locale/resource pages.
```

Avoid:

```text
This is great for SEO and should help performance.
```

## Data Integrity

Use cautious wording when evidence is an example or sample:

- "sample linked evidence"
- "returned evidence rows"
- "materialized projection value"
- "projection considers the issue resolved"
